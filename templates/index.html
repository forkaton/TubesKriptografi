Pengujian Keamanan Kriptografi

Avalanche SHA-3-256

SAC Test
Rumus Perhitungan
1. Untuk iter i: m' = flip 1 bit dari m 2. Δᵢ = HammingDistance( H(m), H(m') ) 3. pᵢ = (Δᵢ / 256) × 100% 4. mean = (1/N) · Σ pᵢ 5. std = √( (1/N) · Σ(pᵢ − mean)² ) 6. SAC PASS ⇔ 45% ≤ mean ≤ 55%
Strict Avalanche Criterion: perubahan 1 bit input harus mengubah ±50% bit output.

Avalanche AES-256-GCM

Key Sensitivity
Rumus Perhitungan
1. K₁ ← random 256-bit; K₂ = K₁ ⊕ flip 1 bit 2. tag₁ = AES-GCM(K₁, IV, m).tag 3. tag₂ = AES-GCM(K₂, IV, m).tag 4. Δᵢ = HammingDistance(tag₁, tag₂) // 128 bit 5. pᵢ = (Δᵢ / 128) × 100% 6. mean, std seperti Avalanche SHA-3
Mengukur sensitivitas key — flip 1 bit kunci harus mengubah ±50% bit auth tag.

Collision Resistance

SHA-3 Hash
Rumus Perhitungan
1. Bangkitkan N pasang pesan acak {mᵢ} 2. hᵢ = SHA-3-256(mᵢ) 3. C = |{ (i,j) : hᵢ = hⱼ, i ≠ j }| 4. Birthday bound: P(collision) ≈ N² / (2 · 2²⁵⁶) 5. PASS ⇔ C = 0
Sebab keamanan koliisi SHA-3-256 ≈ 2¹²⁸ — secara praktis mustahil ditemukan secara acak.

Performance Benchmark

AES Throughput
Rumus Perhitungan
Untuk size ∈ {50,100,500,1000,5000} char: 1. Ulangi R kali, ukur encᵢ & decᵢ (ms) 2. enc_ms = (1/R) · Σ encᵢ 3. dec_ms = (1/R) · Σ decᵢ 4. size_MB = size / (1024·1024) 5. throughput = size_MB / (enc_ms / 1000) 6. PASS ⇔ enc_ms < 5 ms
Mengukur kecepatan enkripsi/dekripsi AES-256-GCM untuk berbagai ukuran pesan.
